name: Customer Journey
description: Jornada do usuário em 5+1 fases (Consciência / Interesse / Avaliação / Compra / Experiência / Retenção) com mapeamento de touchpoints por categoria EPOMS, análise emocional por fase e personas orientadas a dados. Identifica os pontos de abandono e os momentos de encantamento para otimizar cada etapa da experiência do cliente.
---

Map and improve the customer journey for this product.

1. PERSONA DEFINITION — Based on the product, define 1-2 primary user personas. For each: demographics (role, age range, tech comfort), primary goal when using this product, top 3 frustrations with current alternatives, success metric (what does "this product worked" mean for them)?

2. JOURNEY MAPPING — Map the full journey: Awareness → Consideration → Decision → Onboarding → Regular Use → Advocacy. For each phase: What is the user trying to accomplish? What are the pain points? What does the product currently do (or not do) at this phase?

3. ONBOARDING AUDIT — The most critical phase. Check: Is there an empty state that guides first-time users? Is there a progress indicator for multi-step onboarding? Is the "aha moment" (first time user gets value) reached within the first session? How many steps until the user can complete their first meaningful task?

4. RETENTION MOMENTS — Identify 3 moments in the product where users are most likely to disengage (e.g., after day 1, after first failed task, after hitting a paywall). For each: what is currently happening, and what could be done to retain the user?

5. TOUCHPOINTS OUTSIDE THE APP — Map touchpoints beyond the product: onboarding emails, notification strategy, in-app announcements, support interactions, upgrade prompts. Which are currently missing or could be improved?

6. SUCCESS METRICS — For each journey phase, define one measurable metric: Activation rate, Day-1 retention, Day-7 retention, Feature adoption rate, NPS. What data is currently tracked vs. what should be?

Provide a journey map overview and 3 high-priority improvements.
