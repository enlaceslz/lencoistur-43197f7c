name: Design Process
description: Sete fases do processo de design do briefing ao handoff: Briefing → Estratégia → Wireframe → Design Visual → Prototipagem → Revisão → Entrega. Enfatiza clareza nos requisitos antes de criar qualquer pixel e refinamento iterativo com stakeholders a cada etapa, reduzindo retrabalho e garantindo que o resultado final atende ao objetivo de negócio.
---

Review and improve the design process for this project.

Audit where the project is in the design process and identify gaps:

1. BRIEFING — Is there a clear project brief? Does it include: target audience, primary user goal, business goal, constraints (tech, time, budget), success metrics, and competitive references? If not, create a one-page brief now.

2. RESEARCH — What user research has been done? (User interviews, surveys, analytics review, competitor analysis). If none, identify the top 3 assumptions that most need validation before continuing development.

3. INFORMATION ARCHITECTURE — Has the site/app structure been mapped (sitemap, user flows, data model)? If not, create a sitemap for the main sections and a flow diagram for the top user task.

4. WIREFRAMES — Were low-fidelity wireframes created and reviewed before visual design? If the project went straight to high-fidelity, identify the 2-3 screens that most need wireframe-level reconsideration.

5. VISUAL DESIGN — Does a visual design system exist (see ui-design skill)? Are there approved mockups for all major screen states (empty, loading, error, success)?

6. PROTOTYPING — Is there an interactive prototype for user testing? If not, identify the most complex user flow that needs a prototype for validation.

7. HANDOFF & DOCUMENTATION — Is there developer documentation for: component specs (spacing, colors, font sizes), interaction specifications (animation timing, hover states), and edge cases (empty states, error states, loading states)?

Provide a design process assessment and prioritized recommendations for what to do next.
