Apply professional UI design principles to this project.

Analyze the current UI and provide specific, actionable improvements for:

1. LAYOUT & GRID — Is there a consistent grid system (12-column or 4-column mobile)? Are margins and gutters consistent? Flag any elements that break the grid.

2. VISUAL HIERARCHY — Is there a clear primary → secondary → tertiary information hierarchy? Check: font sizes, font weights, color contrast between levels, spacing between elements.

3. SPACING SYSTEM — Is spacing based on a consistent scale (4px, 8px, 16px, 24px, 32px, 48px, 64px)? Flag any arbitrary spacing values.

4. COMPONENT CONSISTENCY — Are buttons, inputs, cards, and badges visually consistent across the app? Check border-radius, padding, shadow, and color usage.

5. DESIGN TOKENS — Are colors, typography, and spacing defined as CSS variables or Tailwind config tokens? Or are they hardcoded throughout the codebase?

6. STYLEGUIDE — Does a styleguide or design system exist? If not, identify the 10 most-used components and suggest defining them as reusable primitives.

For each issue: identify the specific component/page, explain the problem, and provide the corrected CSS or component code.
